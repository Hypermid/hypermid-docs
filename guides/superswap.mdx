Swap any token on any chain to PulseChain tokens (PLS, HEX, PLSX) — and back. Uses the same endpoints as every other Hypermid swap.

## Endpoints You'll Use

| Step | Endpoint | Purpose |
|------|----------|---------|
| 1 | [`GET /v1/quote`](/api-reference/quote) | Get the swap price and route |
| 2 | [`POST /v1/execute`](/api-reference/execute) | Get transaction data to sign |
| 3 | [`GET /v1/status`](/api-reference/status) | Track swap progress |
| 3b | [`POST /v1/inbound-receiver/register`](/api-reference/confirm-deposit) | Confirm deposit (only some routes — see below) |

## Quick Integration (5 minutes)

Copy this code. It handles all PulseChain swap types.

<CodeGroup>
```typescript TypeScript
import { Hypermid } from "@hypermid/sdk";

const client = new Hypermid({ apiKey: "your-api-key" });

// 1. Get a quote
const quote = await client.getQuote({
  fromChain: 8453,          // Base
  toChain: 369,             // PulseChain
  fromToken: "0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913", // USDC on Base
  toToken: "0xA1077a294dDE1B09bB078844df40758a5D0f9a27",   // PLS
  fromAmount: "5000000",    // $5 USDC
  fromAddress: "0xYourWallet",
  slippage: 0.03,
});

console.log("You'll receive:", quote.data.estimate.toAmount, "PLS");
console.log("Estimated time:", quote.data.estimate.executionDuration, "seconds");

// 2. Sign and broadcast
const tx = await wallet.sendTransaction(quote.data.transactionRequest);
console.log("Tx hash:", tx.hash);

// 3. Track status
let status;
do {
  await new Promise(r => setTimeout(r, 10000)); // wait 10s
  status = await client.getStatus({ txHash: tx.hash, fromChain: 8453, toChain: 369 });
  console.log("Status:", status.data.status);
} while (status.data.status === "PENDING");

console.log("Done! Received:", status.data.receiving?.amount);
```

## Standard Integration (30 minutes)

Add error handling, handle both response types, and confirm deposits when needed.

```typescript
async function swapToPulseChain(params: {
  fromChain: number;
  fromToken: string;
  toToken: string;
  fromAmount: string;
  fromAddress: string;
}) {
  // 1. Get quote
  const quote = await client.getQuote({
    ...params,
    toChain: 369,
    slippage: 0.03,
  });

  if (!quote.data) throw new Error(quote.error?.message || "No route found");

  // 2. Sign the transaction
  const txRequest = quote.data.steps?.[0]?.transactionRequest
    || quote.data.transactionRequest;

  const txHash = await wallet.sendTransaction(txRequest);
  await publicClient.waitForTransactionReceipt({ hash: txHash });

  // 3. Confirm deposit (only if the quote says to)
  if (quote.data.afterStep1) {
    const signature = await wallet.signTypedData({
      ...quote.data.afterStep1.eip712,
      message: { ...quote.data.afterStep1.eip712.message, txHash },
    });

    const res = await fetch(quote.data.afterStep1.action, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ ...quote.data.afterStep1.body, txHash, signature }),
    });

    if (!res.ok) throw new Error("Deposit confirmation failed");
  }

  // 4. Poll status
  while (true) {
    const status = await client.getStatus({
      txHash,
      fromChain: params.fromChain,
      toChain: 369,
    });

    if (status.data.status === "DONE") return status.data;
    if (status.data.status === "FAILED") throw new Error("Swap failed");

    await new Promise(r => setTimeout(r, 10000));
  }
}
```

<Tip>
  The `afterStep1` field only appears when a confirmation is needed (non-stablecoin → PulseChain routes). For USDC input or outbound swaps, it's not present — you just sign one transaction.
</Tip>

## Advanced Integration (1 hour)

### Add Webhooks

Get notified when swaps complete instead of polling:

```typescript
// Create a webhook
await fetch("https://api.hypermid.io/v1/partner/webhooks", {
  method: "POST",
  headers: { "Content-Type": "application/json", "X-API-Key": "your-api-key" },
  body: JSON.stringify({
    url: "https://your-server.com/webhook",
    events: ["swap.completed", "swap.failed"],
  }),
});
```

See the [Webhooks guide](/guides/webhooks) for full setup.

### Handle Deposit-Based Routes

If the user is swapping from a non-EVM chain (XRP, Dogecoin, etc.) to PulseChain, the execute response may include a `depositAddress` instead of a `transactionRequest`. See [Execute Swap](/api-reference/execute) for how to handle both shapes.

### Track Detailed Status

PulseChain swaps have additional status detail:

| Status | SubStatus | Description |
|--------|-----------|-------------|
| `PENDING` | `WAIT_SOURCE_CONFIRMATIONS` | Transaction submitted |
| `PENDING` | `BRIDGE_IN_PROGRESS` | Cross-chain transfer (~3-5 min) |
| `PENDING` | `SWAP_IN_PROGRESS` | DEX swap on destination |
| `DONE` | `COMPLETED` | Tokens delivered |
| `DONE` | `FALLBACK_SENT` | Swap unavailable — stablecoin returned |
| `FAILED` | `BRIDGE_TIMEOUT` | Bridge didn't complete in 30 min |
