# Near Intents

Near Intents is HyperMid's intent-based transfer system that enables swaps involving non-EVM chains like NEAR, TON, Tron, XRP, Dogecoin, Litecoin, and others. Unlike traditional bridge transactions that require on-chain smart contract calls, Near Intents uses a deposit-based model where users send tokens to a temporary deposit address.

## Supported Chains

Near Intents chains use chain IDs starting at `900000000`:

| Chain | Chain ID | Deposit Mode |
|-------|----------|-------------|
| NEAR | 900000001 | Manual |
| TON | 900000002 | Wallet |
| Tron | 900000003 | Wallet |
| XRP | 900000004 | Manual |
| DOGE | 900000005 | Manual |
| LTC | 900000006 | Manual |
| BCH | 900000007 | Manual |
| Stellar | 900000008 | Manual |
| Cardano | 900000009 | Manual |
| Aptos | 900000010 | Manual |
| StarkNet | 900000011 | Manual |
| Dash | 900000012 | Manual |
| Zcash | 900000013 | Manual |
| Aleo | 900000014 | Manual |
| ADI | 900000015 | Manual |

<Info>
  Near Intents can also handle deposits from EVM chains, Solana, and Bitcoin. In these cases the deposit mode is "wallet" because a transaction request can be constructed.
</Info>

## Deposit Modes

There are two ways users can deposit tokens for a Near Intents swap:

### Wallet Mode

Used when the user has a connected wallet that can programmatically sign and send transactions (EVM, Solana, Bitcoin, TON, Tron).

In wallet mode, the execute response includes a `transactionRequest` object that you can pass directly to the wallet for signing:

```typescript
const exec = await client.execute({
  fromChain: 1,        // Ethereum (EVM — wallet mode)
  toChain: 900000004,  // XRP (Near Intents destination)
  fromToken: "0x0000000000000000000000000000000000000000",
  toToken: "XRP",
  fromAmount: "1000000000000000000",
  fromAddress: "0xYourEthAddress",
  toAddress: "rYourXRPAddress",
});

// Wallet mode: sign and send the transaction
if (exec.data.transactionRequest) {
  const tx = await wallet.sendTransaction(exec.data.transactionRequest);
  await tx.wait();
}
```

### Manual Mode

Used when the source chain doesn't support programmatic transaction construction (NEAR, XRP, DOGE, LTC, BCH, Stellar, Cardano, Aptos, etc.).

In manual mode, the execute response provides a `depositAddress` (and optionally a `memo`) that you display to the user. The user must manually send tokens to this address using their native wallet.

```typescript
const exec = await client.execute({
  fromChain: 900000004,  // XRP (source)
  toChain: 42161,        // Arbitrum (destination)
  fromToken: "XRP",
  toToken: "0xaf88d065e77c8cC2239327C5EDb3A432268e5831",
  fromAmount: "50000000",  // 50 XRP in drops
  fromAddress: "rYourXRPAddress",
  toAddress: "0xYourEthAddress",
});

// Manual mode: display deposit info to user
if (exec.data.depositAddress && !exec.data.transactionRequest) {
  console.log("Please send your tokens to:");
  console.log("Address:", exec.data.depositAddress);
  if (exec.data.memo) {
    console.log("Memo:", exec.data.memo);
  }
  console.log("Deposit ID:", exec.data.depositId);
}
```

<Warning>
  For chains that require a memo (like XRP and Stellar), the memo is mandatory. If the user sends tokens without the correct memo, the deposit may not be detected and funds could be lost.
</Warning>

## Overriding Deposit Mode

Partners can override the deposit mode to `"manual"` when the user hasn't connected a wallet. This is useful for displaying QR codes or deposit instructions instead of requiring a wallet connection.

```typescript
const exec = await client.execute({
  fromChain: 1,
  toChain: 42161,
  fromToken: "0x0000000000000000000000000000000000000000",
  toToken: "0xaf88d065e77c8cC2239327C5EDb3A432268e5831",
  fromAmount: "1000000000000000000",
  fromAddress: "0xUserAddress",
  toAddress: "0xUserAddress",
  depositMode: "manual",  // Force manual mode even for EVM
});

// Will always return depositAddress, no transactionRequest
console.log("Deposit address:", exec.data.depositAddress);
```

## Auto-Detection

By default, HyperMid automatically determines the deposit mode based on the source chain:

- **EVM, Solana, Bitcoin, TON, Tron** — defaults to `wallet`
- **NEAR, XRP, DOGE, LTC, BCH, Stellar, Cardano, Aptos, StarkNet, Dash, Zcash, Aleo, ADI** — always `manual`

You can check the mode in the execute response:

```typescript
console.log("Deposit mode:", exec.data.depositMode);
// "wallet" or "manual"
```

## Submitting a Manual Deposit

After the user sends tokens manually, you should notify HyperMid that the deposit has been made:

```typescript
const submitResult = await client.submitDeposit({
  depositId: exec.data.depositId,
  txHash: "user-provided-transaction-hash", // Optional
});
```

```bash
curl -X POST "https://api.hypermid.io/v1/execute/deposit/submit" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key" \
  -d '{
    "depositId": "dep_abc123",
    "txHash": "user-provided-transaction-hash"
  }'
```

## Tracking Near Intents Status

Use the deposit status endpoint to track the progress of a Near Intents swap:

<CodeGroup>
```typescript TypeScript
async function pollDepositStatus(depositId: string) {
  while (true) {
    const status = await client.getDepositStatus({ depositId });

    console.log("Status:", status.data.status);
    console.log("Sub-status:", status.data.subStatus);

    switch (status.data.status) {
      case "PENDING":
        console.log("Waiting for deposit...");
        break;
      case "DEPOSIT_RECEIVED":
        console.log("Deposit confirmed, processing swap...");
        break;
      case "DONE":
        console.log("Swap complete!");
        console.log("Output tx:", status.data.destinationTxHash);
        return status.data;
      case "FAILED":
        console.error("Swap failed:", status.data.message);
        throw new Error(status.data.message);
    }

    await new Promise((r) => setTimeout(r, 15000)); // Poll every 15 seconds
  }
}
```

```bash cURL
curl "https://api.hypermid.io/v1/execute/deposit/status?depositId=dep_abc123" \
  -H "X-API-Key: your-api-key"
```
</CodeGroup>

## Complete Example: XRP to USDC on Arbitrum

```typescript
import { HyperMid } from "@hypermid/sdk";

const client = new HyperMid({ apiKey: process.env.HYPERMID_API_KEY });

async function xrpToUsdc() {
  // 1. Get a quote
  const quote = await client.getQuote({
    fromChain: 900000004,  // XRP
    toChain: 42161,        // Arbitrum
    fromToken: "XRP",
    toToken: "0xaf88d065e77c8cC2239327C5EDb3A432268e5831",
    fromAmount: "50000000", // 50 XRP
    fromAddress: "rYourXRPAddress",
  });

  console.log("Estimated USDC output:", quote.data.estimate.toAmount);

  // 2. Execute — will be manual mode for XRP
  const exec = await client.execute({
    fromChain: 900000004,
    toChain: 42161,
    fromToken: "XRP",
    toToken: "0xaf88d065e77c8cC2239327C5EDb3A432268e5831",
    fromAmount: "50000000",
    fromAddress: "rYourXRPAddress",
    toAddress: "0xYourEthAddress",
  });

  // 3. Display deposit instructions to user
  console.log("=== Send XRP to complete the swap ===");
  console.log("Address:", exec.data.depositAddress);
  console.log("Memo:", exec.data.memo);
  console.log("Amount: 50 XRP");
  console.log("=====================================");

  // 4. After user sends, submit the deposit
  const txHash = await promptUserForTxHash(); // Your UI logic
  await client.submitDeposit({
    depositId: exec.data.depositId,
    txHash,
  });

  // 5. Poll for completion
  let status;
  do {
    await new Promise((r) => setTimeout(r, 15000));
    status = await client.getDepositStatus({
      depositId: exec.data.depositId,
    });
    console.log("Status:", status.data.status);
  } while (!["DONE", "FAILED"].includes(status.data.status));

  if (status.data.status === "DONE") {
    console.log("Swap complete! USDC received on Arbitrum.");
  }
}
```

## Best Practices

1. **Always display the memo** — For chains that require it (XRP, Stellar), prominently display the memo and warn users that omitting it may result in lost funds.

2. **Set deposit expiration UI** — Deposit addresses are time-limited. Show a countdown or expiration warning to the user.

3. **Poll at reasonable intervals** — Near Intents swaps can take 1-5 minutes. Poll every 15 seconds to avoid unnecessary API calls.

4. **Handle partial deposits** — If a user sends a different amount than quoted, the system will still attempt to process the swap with the received amount.

5. **Use webhooks for production** — Instead of polling, set up [webhooks](/guides/webhooks) to receive status updates in real time.
