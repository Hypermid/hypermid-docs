<Tip>
  You can also fetch this list programmatically via [GET /v1/chains](/api-reference/chains).
</Tip>

### Deposit Mode Explained

- **Wallet** — The execute endpoint returns a `transactionRequest` that can be signed and sent programmatically. Supported for EVM chains, Solana, Bitcoin, TON, and Tron as source chains.
- **Manual** — The execute endpoint returns a `depositAddress` (and optional `memo`) that the user must send tokens to using their native wallet.

<Note>
  Partners can override the deposit mode to "manual" for any chain using the `depositMode` parameter in the execute request. See the [Near Intents Guide](/guides/near-intents) for details.
</Note>

## Chain ID Lookup

Use the chain ID when calling these endpoints:

- `GET /v1/tokens?chains=1,42161` — get tokens by chain
- `GET /v1/quote?fromChain=1&toChain=42161` — get a swap quote
- `POST /v1/execute` — execute a swap
- `GET /v1/gas-prices?chainId=1` — get gas prices
- `GET /v1/status?fromChain=1&toChain=42161` — check swap status
