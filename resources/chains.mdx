Hypermid supports 90+ blockchains across EVM, non-EVM, and Near Intents networks.

## Unified Chain Identifiers

Every chain has a **slug** (human-readable string) and a **numeric ID**. Both are accepted everywhere — API params, widget config, and SDK methods.

<Tip>
  **Use slugs for non-EVM chains** — `"solana"` is much easier than `1151111081099710`. EVM chains work with either format.
</Tip>

You can also fetch the full list programmatically via [GET /v1/chains](/api-reference/chains) — each chain object includes `slug`, `providers`, and `id`.

### Deposit Mode Explained

- **Wallet** — The execute endpoint returns a `transactionRequest` that can be signed and sent programmatically. Supported for EVM chains, Solana, Bitcoin, TON, and Tron as source chains.
- **Manual** — The execute endpoint returns a `depositAddress` (and optional `memo`) that the user must send tokens to using their native wallet.

<Note>
  Partners can override the deposit mode to "manual" for any chain using the `depositMode` parameter in the execute request. See the [Near Intents Guide](/guides/near-intents) for details.
</Note>

## Using Chain Identifiers

Both slugs and numeric IDs work in all API endpoints:

```bash
# Using numeric ID
curl "https://api.hypermid.io/v1/quote?fromChain=1&toChain=42161&..."

# Using slug (recommended for non-EVM)
curl "https://api.hypermid.io/v1/quote?fromChain=ethereum&toChain=solana&..."
```

```javascript
// Widget config — both formats work
HypermidWidget.init({
  defaultFromChain: "solana",     // slug
  defaultToChain: 42161,          // numeric ID
});
```

```typescript
// SDK — both formats work
import { ChainSlug, ChainId } from "@hypermid/sdk";

const quote = await hm.getQuote({
  fromChain: ChainSlug.SOLANA,    // "solana"
  toChain: ChainId.ARBITRUM,      // 42161
  ...
});
```
