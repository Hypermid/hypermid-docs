---
title: "Register Inbound Deposit"
openapi: "POST /v1/inbound-receiver/register"
description: "Register a verified USDC deposit for automatic bridging to PulseChain"
---

After a multi-step inbound SuperSwap completes Step 1 (LiFi swap), the frontend must call this endpoint to register the deposit. The backend verifies the transaction on-chain and initiates the automatic bridge to PulseChain.

**This endpoint requires an EIP-712 wallet signature** from the deposit sender (`fromAddress`), proving they authorized the specific routing parameters. This prevents attackers from redirecting deposits.

## Flow

1. User gets a multi-step SuperSwap quote from [`GET /v1/quote`](/api-reference/quote)
2. User signs and broadcasts the LiFi transaction (Step 1)
3. Frontend waits for transaction confirmation on Base
4. Frontend prompts the user to sign an **EIP-712 message** (gasless — no tx needed)
5. Frontend calls this endpoint with the tx hash + signature
6. Backend verifies the tx on-chain, creates a record, and starts automatic bridging (~5 min)

<ParamField body="txHash" type="string" required>
  The confirmed LiFi transaction hash from Step 1 (on Base).
</ParamField>

<ParamField body="fromAddress" type="string" required>
  The sender wallet address. Must match `tx.from` of the submitted transaction.
</ParamField>

<ParamField body="toAddress" type="string">
  Recipient wallet address on PulseChain. Defaults to `fromAddress` if not provided.
</ParamField>

<ParamField body="outputToken" type="string" required>
  Desired output token address on PulseChain. Use `0xA1077a294dDE1B09bB078844df40758a5D0f9a27` for WPLS (PLS).
</ParamField>

<ParamField body="destinationDomain" type="number">
  Hyperlane destination domain. Default: `369` (PulseChain).
</ParamField>

<ParamField body="signature" type="string" required>
  EIP-712 signature from the sender wallet, authorizing `{txHash, toAddress, outputToken, destinationDomain}`.
</ParamField>

## EIP-712 Signing

The wallet must sign a typed data message with this structure:

```json
{
  "domain": {
    "name": "HypermidInboundReceiver",
    "version": "1",
    "chainId": 8453
  },
  "types": {
    "RegisterDeposit": [
      { "name": "txHash", "type": "bytes32" },
      { "name": "toAddress", "type": "address" },
      { "name": "outputToken", "type": "address" },
      { "name": "destinationDomain", "type": "uint32" }
    ]
  },
  "primaryType": "RegisterDeposit",
  "message": {
    "txHash": "0x<your-step1-tx-hash>",
    "toAddress": "0x<recipient-on-pulsechain>",
    "outputToken": "0xA1077a294dDE1B09bB078844df40758a5D0f9a27",
    "destinationDomain": 369
  }
}
```

<CodeGroup>
```typescript viem
import { createWalletClient, custom } from 'viem';
import { base } from 'viem/chains';

const walletClient = createWalletClient({
  chain: base,
  transport: custom(window.ethereum),
});

// 1. Sign the EIP-712 message (gasless)
const signature = await walletClient.signTypedData({
  domain: {
    name: "HypermidInboundReceiver",
    version: "1",
    chainId: 8453,
  },
  types: {
    RegisterDeposit: [
      { name: "txHash", type: "bytes32" },
      { name: "toAddress", type: "address" },
      { name: "outputToken", type: "address" },
      { name: "destinationDomain", type: "uint32" },
    ],
  },
  primaryType: "RegisterDeposit",
  message: {
    txHash: step1TxHash,               // from Step 1
    toAddress: recipientAddress,        // or sender address
    outputToken: "0xA1077a294dDE1B09bB078844df40758a5D0f9a27",
    destinationDomain: 369,
  },
});

// 2. Register the deposit
const response = await fetch("https://api.hypermid.io/v1/inbound-receiver/register", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    txHash: step1TxHash,
    fromAddress: senderAddress,
    toAddress: recipientAddress,
    outputToken: "0xA1077a294dDE1B09bB078844df40758a5D0f9a27",
    destinationDomain: 369,
    signature,
  }),
});

const result = await response.json();
// result.data.status === "USDC_ARRIVED" → bridge starts automatically
```

```bash cURL
curl -X POST "https://api.hypermid.io/v1/inbound-receiver/register" \

    "destinationDomain": 369,
    "signature": "0x..."
  }'
```
